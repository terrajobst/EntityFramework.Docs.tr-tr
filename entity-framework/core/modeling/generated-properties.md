<span data-ttu-id="c8451-113">Bir varlık özelliğine atanan bir değere sahip bağlam eklerseniz, yeni bir tane oluşturmak yerine bu değeri eklemek EF deneyecek.</span><span class="sxs-lookup"><span data-stu-id="c8451-113">If you add an entity to the context that has a value assigned to the property, then EF will attempt to insert that value rather than generating a new one.</span></span> <span data-ttu-id="c8451-114">Bir özelliğin CLR varsayılan değer atanmamışsa atanmış bir değere sahip olduğu kabul edildiği (`null` için `string`, `0` için `int`, `Guid.Empty` için `Guid`, vb..).</span><span class="sxs-lookup"><span data-stu-id="c8451-114">A property is considered to have a value assigned if it is not assigned the CLR default value (`null` for `string`, `0` for `int`, `Guid.Empty` for `Guid`, etc.).</span></span> <span data-ttu-id="c8451-115">Daha fazla bilgi için bkz: [oluşturulan özellikler için açık değerler](..\saving\explicit-values-generated-properties.md).</span><span class="sxs-lookup"><span data-stu-id="c8451-115">For more information, see [Explicit values for generated properties](..\saving\explicit-values-generated-properties.md).</span></span>
